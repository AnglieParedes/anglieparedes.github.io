Paso 01 de 06
Instalar Node.js
Node.js es el motor que necesitas para ejecutar React y Vite en tu computador. Es gratuito y oficial.
Descargar e instalar

Ve a nodejs.org y descarga la versión LTS (la recomendada). Es la que dice "Recommended For Most Users".

Instala normalmente haciendo clic en "Next" en todo. No necesitas cambiar ninguna opción.

Verificar la instalación

Abre la Terminal (en Mac: busca "Terminal" en Spotlight. En Windows: busca "cmd" o "PowerShell") y escribe:

node --version # Debería mostrar algo como: v20.15.0 npm --version # Debería mostrar algo como: 10.7.0

Si ves números de versión, ¡perfecto! Node.js está instalado correctamente.

Tip: Si ya tienes Node instalado con una versión anterior a v18, actualízalo descargando la versión LTS nuevamente desde nodejs.org.

Paso 02 de 06
Crear cuenta en GitHub
GitHub es donde vas a guardar tu código y publicar tu web gratis. Es como el Google Drive del código.
Crear la cuenta

Ve a github.com y créate una cuenta con tu email. El nombre de usuario importa — será parte de tu URL.

Importante: Elige un username profesional. Lo ideal es anglieparedes o anglie-paredes. Tu portafolio quedará en: anglieparedes.github.io
Crear el repositorio

Una vez con tu cuenta, crea un nuevo repositorio (repo = carpeta de proyecto en GitHub):

  • 1. Haz clic en el botón verde "New" o el ícono "+"
  • 2. En "Repository name" escribe exactamente: anglieparedes.github.io
  • 3. Márcalo como Public
  • 4. Haz clic en "Create repository"
El nombre del repo es clave: Si tu usuario es anglieparedes, el repo debe llamarse anglieparedes.github.io para que funcione como dominio principal gratuito.
Instalar Git en tu computador

Git es la herramienta para enviar tu código a GitHub. Descárgala en git-scm.com e instálala con todas las opciones por defecto.

# Verifica que Git está instalado: git --version # Resultado: git version 2.x.x

Paso 03 de 06
Instalar y probar el proyecto
Descarga la carpeta del proyecto que te entregué, instala las dependencias y míralo en tu computador antes de publicarlo.
Descomprimir el proyecto

Descarga el archivo anglie-portfolio.zip que te entregué y descomprímelo en una carpeta fácil de encontrar, por ejemplo en tu Escritorio o en Documentos.

Abrir la terminal en la carpeta del proyecto

Mac: Arrastra la carpeta al ícono de Terminal, o abre Terminal y escribe cd (con espacio) y arrastra la carpeta.

Windows: Abre la carpeta en el Explorador de Archivos, haz clic derecho en un espacio vacío y elige "Abrir en Terminal".

# Verificar que estás en la carpeta correcta: ls # Deberías ver: package.json src/ index.html vite.config.js
Instalar dependencias y ejecutar
# 1. Instalar React, Vite y Framer Motion (solo la primera vez) npm install # 2. Ver el portafolio en tu navegador npm run dev

Después de npm run dev verás algo como:

VITE v5.x.x ready in 320 ms ➜ Local: http://localhost:5173/ ➜ Network: http://192.168.x.x:5173/

Abre http://localhost:5173 en tu navegador y verás tu portafolio. 🎉

Para detener el servidor: En la terminal presiona Ctrl + C

Paso 04 de 06
Subir el código a GitHub
Ahora conectamos tu carpeta local con el repo de GitHub y subimos el código. Solo necesitas hacer esto una vez.
Configurar Git (solo la primera vez)
# Reemplaza con tu nombre y email de GitHub git config --global user.name "Anglie Paredes" git config --global user.email "aeparedest@gmail.com"
Subir el proyecto
# 1. Inicializar Git en la carpeta git init # 2. Agregar todos los archivos git add . # 3. Primer commit (guardar los cambios) git commit -m "✨ portafolio inicial" # 4. Conectar con tu repo de GitHub # (reemplaza 'anglieparedes' con tu usuario) git remote add origin https://github.com/anglieparedes/anglieparedes.github.io.git # 5. Subir el código git push -u origin main

GitHub te pedirá tu usuario y contraseña (o un token personal si tienes 2FA activado).

Para el futuro: Cada vez que cambies algo en el código, solo necesitas ejecutar:
git add . && git commit -m "descripción del cambio" && git push

Paso 05 de 06
Activar GitHub Pages
Ahora activamos el deploy automático. Cada vez que subas código, tu web se actualiza sola gracias al workflow que ya viene incluido en el proyecto.
Activar GitHub Actions Pages

En tu repo de GitHub, ve a:

  • 1. Haz clic en Settings (pestaña arriba a la derecha)
  • 2. En el menú izquierdo, busca Pages
  • 3. En "Source" selecciona GitHub Actions
  • 4. Guarda los cambios
Ver el deploy en acción

Ve a la pestaña Actions de tu repo. Verás un workflow corriendo (círculo amarillo). Cuando termine y se ponga verde ✅, tu web está publicada.

El primer deploy tarda ~2-3 minutos. Los siguientes son más rápidos.

Cómo actualizar contenido

Todo el contenido editable está en los archivos .jsx dentro de src/components/. Cada sección tiene su propio archivo:

# Proyectos → edita Projects.jsx (el array 'projects') # Experiencia → edita Experience.jsx (el array 'jobs') # Educación → edita Education.jsx (el array 'items') # Planes RRSS → edita Services.jsx (el array 'plans') # Contacto → edita Contact.jsx (el array 'contactItems')
Para conectar el formulario de verdad (EmailJS)
# 1. Instalar EmailJS npm install @emailjs/browser # 2. Crea cuenta gratuita en emailjs.com # 3. En Contact.jsx descomenta las líneas de emailjs # 4. Reemplaza SERVICE_ID, TEMPLATE_ID y PUBLIC_KEY
Para agregar dominio .cl propio (futuro)
# 1. Compra anglieparedes.cl en nic.cl (~$10.000/año) # 2. Crea cuenta gratuita en cloudflare.com # 3. En Cloudflare: agrega tu dominio y apunta DNS a GitHub # 4. En GitHub Pages settings: agrega custom domain # 5. En vite.config.js: cambia base a '/' ← ya está listo
Flujo de trabajo para siempre:
Editas código → git add . && git commit -m "cambio" && git push → GitHub Pages actualiza tu web automáticamente en ~2 minutos.
